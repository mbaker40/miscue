# P2 — Act-based wave generation (`src/waves.ts`)

**Files you own:** `src/waves.ts` only. Touch nothing else.
**Read first:** `docs/tasks/README.md`, current `src/waves.ts`, `src/entities.ts` (table constants),
`src/physics.ts` (POCKETS export).

## Goal / dossier clauses

D3: *"Three to four waves … escalating by composition (combining enemy types into nasty interactions) and
pressure (spawning near corners/pockets) far more than by raw count."* Every rack is *"a mini three-act:
teach (one clean enemy type) → complicate (nasty combinations) → Elite climax."*
D5 (8-ball): *"a full table of Elites."*

## Current behavior (to replace)

- `nWaves = depth <= 1 ? 2 : 3` — table 1 gets only 2 waves.
- Teach wave only when `w === 0 && depth <= 2` — no teach beat from depth 3 on.
- `eliteBudget = money ? 2 : 1` — flat, no act scaling.
- `place()` always spawns in the far half, away from pockets — no pressure escalation, ever.
- `makeBossWaves` sizes `[2,3,3]`, only one elite total.

## Required behavior

Acts: depth 1–4 → act 1, 5–8 → act 2, 9–12 → act 3. Compute inline:
`const act = depth <= 4 ? 1 : depth <= 8 ? 2 : 3;` (do NOT import route.ts — keeps waves.ts dependency-free).

### `makeRack(depth, rng, money)` — signature unchanged
- `nWaves = act === 1 ? 3 : 4`.
- Per-wave count: `count = 1 + act + (w > 0 ? 1 : 0) + (money ? 1 : 0)`, capped at 6.
- **Wave 0 is ALWAYS the teach wave**: one clean kind (`rng.pick(pool)`), no elites, at every depth.
- Middle waves: mixed kinds from the pool (current "complication" branch).
- Last wave: mixed + the Elite climax: `eliteBudget = act + (money ? 1 : 0)` elite spawns appended.
- Enemy pool unchanged in shape: bumper+skitter at depth 1, +charger at depth ≥ 2, +splitter at depth ≥ 3
  (full roster by end of Act I; acts 2–3 always use all four).

### Pressure placement
Extend `place` to `place(rng, used, minR, pressure)` where callers pass
`pressure = Math.min(1, (depth - 1) / 11)` (0 at depth 1 → 1 at depth 12).
- With probability `0.6 * pressure`, the spawn is a **pocket-pressure spawn**: pick a random entry from
  `POCKETS` (import from `./physics` — no import cycle; physics does not import waves) and place the spawn
  offset ~`0.12–0.16` (rng.range) from the pocket center, direction biased toward the table interior
  (negate/clamp so the point stays inside `|x| < HALF_W - 0.05`, `|z| < HALF_L - 0.05`, and at least
  `0.09` from the pocket center so nothing spawns inside the capture radius, `POCKET_R = 0.062`).
- Otherwise: the existing far-half placement.
- Keep the existing min-distance retry loop against `used` for both branches.
- `makeBossWaves`/`makeMiniBossWaves` pass `pressure` of `1` and `0.3` respectively.

### `makeBossWaves(rng)` — signature unchanged
- Still exactly **3 waves** (armor 3 is 1:1 with wave clears — invariant #3; do not change wave count).
- Sizes `[2, 2, 3]`, kinds from all four, **every spawn `elite: true`** ("a full table of Elites").
- Keep the existing keep-clear entry for the boss position `{ x: 0, z: -HALF_L * 0.55 }`.

### `makeMiniBossWaves(rng)` — new export
- Exactly **1 wave** (mini-boss armor is 1): 3 enemies from `['bumper', 'charger', 'splitter']`
  (a composition check — "do you have a build yet?"), exactly one of them `elite: true`.
- Keep-clear entry `{ x: 0, z: -HALF_L * 0.5 }` (mini-boss spawn).
