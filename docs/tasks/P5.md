# P5 — Seeded AI streams (D1 replay-readiness)

**Files you own:** `src/rng.ts`, `src/entities.ts`, `src/enemies.ts`, plus exactly ONE line in
`src/game.ts` (`newRun`). Touch nothing else.
**Read first:** `docs/tasks/README.md` (tag constants), the three files above.

## Goal / dossier clause

D1: leaderboard integrity comes from *"logging the player's input stream and replaying top scores
server-side."* Replay only reproduces a run if everything except raw physics is deterministic given the same
inputs. Enemy AI currently rolls unseeded `Math.random()`, so two replays of identical inputs can diverge.
This packet moves AI randomness onto a seeded stream. (Cosmetic `Math.random()` in `render.ts` — crack
texture jitter, camera shake — is explicitly ALLOWED to stay free: it's visual only.)

## Changes

### `src/rng.ts` — module-level AI stream
Append:
```ts
// AI stream (D1): enemy decisions must be replayable. Seeded per run; separate
// from the rack/draft streams so AI consumption can never desync table content.
let aiStream = new Rng(1);
export function seedAiRng(seed: number) { aiStream = new Rng(seed >>> 0); }
export function aiRandom(): number { return aiStream.next(); }
```

### `src/entities.ts`
- Import `aiRandom` from `./rng`.
- In `makeEnemy`: `aiTimer: 0.5 + Math.random() * 1.5` → `aiTimer: 0.5 + aiRandom() * 1.5`.
- (No other `Math.random()` exists in this file; `makePlayer`/`makeSplitChild` need no change.)

### `src/enemies.ts`
- Import `aiRandom` from `./rng`.
- In `interiorTargetAwayFrom`: both `(Math.random() - 0.5) * 0.3` jitters → `(aiRandom() - 0.5) * 0.3`.

### `src/game.ts` — one line only
In `newRun`, right after `this.route = generateRoute(seed);` (or after the seed assignment if that line
reads differently), add:
```ts
seedAiRng((seed ^ AI_TAG) >>> 0);
```
with `seedAiRng` imported from `./rng` and `AI_TAG` from `./route`. Change nothing else in game.ts.

## Out of scope
Do not touch `render.ts` (its `Math.random()` is sanctioned), `physics.ts`, `waves.ts`, `hud.ts`. Do not
reorder or alter any existing `Rng` usage — the rack/draft/route streams must be byte-identical to before.

## Acceptance
1. `npx tsc --noEmit` and `npm run build` pass.
2. `grep -n "Math.random" src/entities.ts src/enemies.ts` → no matches.
3. `grep -n "Math.random" src/render.ts` → matches remain (unchanged file).
4. Determinism spot-check (bundle rng+entities via esbuild): after `seedAiRng(42)`, two runs of
   `[makeEnemy('skitter',0,0).aiTimer, makeEnemy('charger',0,0).aiTimer]` (re-seeding between runs) produce
