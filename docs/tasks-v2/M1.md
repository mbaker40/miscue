# M1 — Combat rack ported

Four packets, two batches. Batch 1: **M1-A ∥ M1-B** (disjoint files). Batch 2: **M1-C ∥ M1-D**.
Ground rules, invariants, and the acceptance floor are in `README.md` — read it first. All v1 references
are to `/src/*.ts` at repo root (READ ONLY — port behavior, never edit or import v1 files).

Shared context for every packet:
- The scaffold commit already landed `v2/src/rng.ts` (Rng/dailySeed/seedAiRng/aiRandom/tags/subRng, v1
  verbatim) and `CFG.combat` / `CFG.enemies` in `v2/src/config.ts`. **Use these constants — do not
  hardcode a number that exists in CFG.** The velocity-scale story: v1 speed thresholds were converted
  ×8/3 (v1 max shot 3.0 → v2 `CFG.shot.maxImpulse` 8); ratios and timers copied verbatim.
- Entities live in `v2/src/registry.ts`: positions/velocities are on `e.body` (Rapier RigidBody) —
  read `body.translation()` / `body.linvel()`, write `body.setLinvel(v, true)`. AI scratch state is
  `e.ai = { timer, state, tx, ty, tz, lungesLeft, didSplit }` (v1's flat aiTimer/aiState/aiTx/aiTz).
  Kill via `despawn(sim, e)` — never bare `alive = false` (the Rapier body must be removed).
- v1 AI state convention carries: `ai.state` 0 = idle, 1 = telegraph/armed, 2 = recovering.

## M1-A — Enemies + waves

**Files you own:** `v2/src/enemies.ts` (new), `v2/src/waves.ts` (new), `v2/src/registry.ts`
(additive only: `spawnSplitChildren`).

**Goal.** Port v1 enemy AI (`/src/enemies.ts`) and wave construction (`/src/waves.ts`) onto Rapier
entities. AI intent per kind is unchanged; actuation becomes `setLinvel` at decision points (v1 set
`vx/vz` directly — same idea, Rapier's damping does the rest). Bumpers and splitters have NO AI
(pure mass); only skitter/charger/boss think.

### enemies.ts — required behavior (v1 logic, CFG numbers)

`export function updateEnemies(enemies: Entity[], player: Entity, dt: number, fx: AiFx, bounds: Bounds): void`
with `export interface AiFx { telegraph(e: Entity, on: boolean): void }` and
`export interface Bounds { halfW: number; halfL: number }` (arena half-extents; roam targets clamp
inside them). Skip `!e.alive`. `playerStill = hypot(player linvel x,z) < CFG.enemies.playerStillSpeed`.

- **skitter** (flee): count down `ai.timer`; when `≤ 0 && playerStill && speedOf(e) < CFG.enemies.enemyStillSpeed`,
  pick an interior target away from the player and `setLinvel` toward it at
  `elite ? eliteSpeed : speed`; reset `ai.timer = elite ? eliteTimerS : timerS`.
  Target = `e.pos + fleeDir * (0.9 * bounds.halfW) + jitter(±0.3 * halfW)` where fleeDir is the unit
  vector away from the player (v1's 0.45 offset on a 0.5-halfW table = 0.9·halfW), clamped to
  `|x| ≤ 0.7·halfW`, `|z| ≤ 0.75·halfL` (v1's clamps). Use `aiRandom()` from rng.ts for jitter.
- **charger** (telegraphed lunge): state 0: when `ai.timer ≤ 0 && playerStill` → state 1,
  `ai.timer = telegraphS (0.75)`, lock `ai.tx/ai.tz = player pos` ("punishes where you STOPPED"),
  `ai.lungesLeft = elite ? 2 : 1`, `fx.telegraph(e, true)`. State 1: when `ai.timer ≤ 0` →
  `fx.telegraph(e, false)`, `setLinvel` toward `(ai.tx, ai.tz)` at `elite ? eliteSpeed : speed`,
  `lungesLeft--`; if `lungesLeft > 0` re-arm (state 1, `ai.timer = relungeS (0.65)`, re-lock target,
  telegraph back on) else state 2, `ai.timer = elite ? eliteRecoverS : recoverS`. State 2: when
  `ai.timer ≤ 0` → state 0, `ai.timer = 0.4`.
- **boss** (same shape, one lunge, slower): state 0: `ai.timer ≤ 0` → state 1, `ai.timer = telegraphS
  (0.9)`, lock target, telegraph on. State 1: fire at `CFG.enemies.boss.speed`, → state 2,
  `ai.timer = e.vulnerable ? recoverVulnS : recoverArmoredS` (a broken boss recovers faster — more
  punishable). State 2: `ai.timer ≤ 0` → state 0, `ai.timer = 0.5`.
  Boss ignores `playerStill` (v1: it lunges on its own clock).
- All lunges/flees are horizontal: preserve the body's current `y` velocity in `setLinvel`.
- New spawns start `ai.timer = CFG.enemies.spawnTimerMin + aiRandom() * CFG.enemies.spawnTimerRand`
  (v1's 0.5 + rand·1.5) — set it inside `spawnWave` (waves.ts) or export a helper; don't touch
  `buildEntity`.

### registry.ts — `spawnSplitChildren` (additive)

`export function spawnSplitChildren(sim: Sim, parent: Entity): Entity[]` — v1 `makeSplitChild` ported:
`n = parent.elite ? 3 : 2` non-elite splitter children at the parent's position, each with
`r = parent.r * 0.68`, `mass = parent.mass * 0.45` (bypass STATS — you'll need a small internal
seam or optional params on the existing private builder; keep `spawnEnemy`'s signature unchanged),
`ai.didSplit = true` (children never re-split), velocity `parentV * 0.5 ± perp * 0.5 * |parentV|`
where `perp = normalize(-vz, vx)` and the sign alternates per child (v1: `i % 2 === 0 ? 1 : -1`).
Caller (M1-C) despawns the parent — this function only spawns.

### waves.ts — required behavior (v1 shapes, normalized coords)

Types: `export interface SpawnDef { kind: Exclude<Kind, 'player'>; nx: number; nz: number; elite: boolean }`
(**normalized** coords in [-1, 1] — the game scales by arena bounds; v1's absolute table coords are
divided by v1 HALF_W/HALF_L when porting the placement math), `export type Wave = SpawnDef[]`.

- `poolFor(depth)`: `['bumper','skitter']`, `+'charger'` at depth ≥ 2, `+'splitter'` at depth ≥ 3.
- `export function makeRack(depth: number, rng: Rng, money: boolean): Wave[]` — v1 verbatim in shape:
  `act = depth <= 4 ? 1 : 2` (**v2 run is 8 tables — do NOT copy v1's act-3/MAX_DEPTH-12 branch**),
  `pressure = min(1, (depth - 1) / 7)` (v1 /11 rescaled to the 8-table run), `nWaves = act === 1 ? 3 : 4`,
  per-wave `count = min(6, 1 + act + (w > 0 ? 1 : 0) + (money ? 1 : 0))`. Wave 0 = teach: ONE kind
  (`rng.pick(pool)`), no elites, minR 0.28 (v1 0.14 on a 0.5-halfW table → normalized ×2). Waves 1+:
  per-spawn `rng.pick(pool)`, no elites, minR 0.26. Last wave additionally gets
  `eliteBudget = act + (money ? 1 : 0)` elite spawns of random kind, minR 0.30.
- Placement `place(rng, used, minR, pressure)` in normalized space: with probability `0.6 * pressure`
  place near a pocket — pockets are the 4 corners `(±1, ±1)` normalized; offset `rng.range(0.24, 0.32)`
  inward along both axes (v1 0.12–0.16 ×2), clamp to `±0.9`, and keep at least `0.18` from the corner
  point (v1's 0.09 capture-radius pushback ×2). Otherwise far-half: `nx = rng.range(-0.72, 0.72)`,
  `nz = rng.range(-0.8, 0.15)` (player spawns at +nz). Rejection-sample vs `used` up to 40 tries at
  `minR`, fallback `nx ∈ [-0.3, 0.3], nz ∈ [-0.7, 0]`.
- `export function makeBossWaves(rng: Rng): Wave[]` — 3 waves, sizes `[2, 2, 3]`, pool
  `['bumper','skitter','charger','splitter']`, EVERY spawn elite, pressure 1, minR 0.28, `used`
  seeded with `{ nx: 0, nz: -0.55 }` (the boss's spot stays clear).
- `export function makeMiniBossWaves(rng: Rng): Wave[]` — single wave of 3, pool
  `['bumper','charger','splitter']`, exactly ONE elite (`eliteIdx = rng.int(0, 2)`), pressure 0.3,
  minR 0.28, `used` seeded `{ nx: 0, nz: -0.5 }`.
- `export function spawnWave(sim: Sim, wave: Wave, bounds: Bounds): Entity[]` — scales normalized
  coords by bounds (spawn y = enemy r + 0.05), calls `spawnEnemy`, seeds each `ai.timer` per the
  spawn-timer rule above.

**Out of scope:** game-state wiring (M1-C owns who calls updateEnemies/spawnWave and all split/crack
triggers), rendering (M1-B implements the real `AiFx`), pocket sensors.

**Acceptance (write to scratchpad, run, then delete nothing — leave scripts in scratchpad):**
1. Floor: `npx tsc --noEmit` + `npm run build` in `v2/`.
2. Wave shapes via esbuild+node (`npx esbuild v2/src/waves.ts --bundle --format=esm --outfile=<scratchpad>/waves.mjs`
   — rng.ts has no DOM deps): across seeds 1..50 — teach wave is single-kind and elite-free at every
   depth 1..8; non-final waves elite-free; final-wave elite count = act (+1 money); boss = [2,2,3] all
   elite; miniboss = one wave of 3 with exactly 1 elite; every spawn within normalized bounds and
   pairwise ≥ minR apart (or fallback-placed); depth-1 pools contain no charger/splitter.
3. AI in node (rapier3d-compat runs in plain node; build a Sim + flat slab + entities, tick manually):
   a skitter with a parked player flees (velocity > 1 after its timer elapses) and its target stays in
   bounds; a charger telegraphs (fx.telegraph(true) observed) for 0.75s then lunges at the player's
   OLD position; `spawnSplitChildren` yields 2 (3 elite) children at r×0.68 with diverging velocities
   that never re-split.

## M1-B — Arena + enemy looks + audio

**Files you own:** `v2/src/arena0.ts` (new), `v2/src/render/balls.ts` (extend), `v2/src/audio.ts` (new).

**Goal.** The combat table M1 boots into, visually distinct enemies with a readable telegraph, and
the v1 SFX kit on the v1 limiter bus.

### arena0.ts

`export function buildArena0(sim: Sim): { spawn: Vec3; checkpoint: Vec3; pockets: { x: number; y: number; z: number; id: number }[]; bounds: { halfW: number; halfL: number } }`

Follow `level0.ts`'s authoring idiom (everything through `sim.addStatic`/`sim.addSensorZone` so
`render/level.ts` auto-builds meshes — that mechanism already exists, do not touch it):
- Main slab 10×10 (`hx: 5, hz: 5`, hy 0.15, top at y=0), floating over the void.
- 4 corner pocket wells: sensor cylinders (halfHeight 0.35, radius 0.45) centered ~0.75u diagonally
  inside each corner, ids 0..3. `render/level.ts` already draws neon rims over pocket sensors.
- Rails (match level0's rail dims: hy 0.14, hz/hx 0.08, chalk-white via the existing trim heuristic)
  along all four edges BUT stopping ~1.4u short of each corner — the corner mouths are open so balls
  can be shot past the pocket into the void OR into the well; a pocket you can't overshoot is no
  pocket at all.
- `spawn = checkpoint = { x: 0, y: 0.2, z: 3.2 }` (player side is +z, matching camera boot yaw π).
- `bounds = { halfW: 5, halfL: 5 }`; pockets array = the 4 sensor world positions with their ids.

### render/balls.ts (extend, keep the existing sync contract)

- Keep KIND_COLOR. Add: **elite look** — elites get a hot rim: set material `emissive` 0xff3ea5,
  `emissiveIntensity` 0.35 (they already read bigger via r).
- **Telegraph FX:** new method `setTelegraph(e: Entity, on: boolean)` storing per-id flag, and inside
  `sync` pulse flagged meshes: `emissive` 0xff4a4a, `emissiveIntensity` 0.4 + 0.35·sin(t·14) (pass
  `performance.now()`-based seconds into sync or keep a module clock). Off restores the kind's base
  emissive. This implements M1-A's `AiFx` — main.ts (M1-C) wires `fx = { telegraph: (e, on) => ballViews.setTelegraph(e, on) }`.
- Boss look: give `boss` a white ring band (thin torus child mesh or second material) so "the 8-ball"
  reads as an 8-ball at a glance. Keep it cheap — one extra mesh on the boss only.

### audio.ts

Port v1 `/src/audio.ts` VERBATIM (it has zero v1-specific imports): lazy AudioContext + resume,
master gain 0.75 → DynamicsCompressor limiter (threshold −12, knee 6, ratio 12, attack 0.002,
release 0.12) → destination, shared 0.5s noise buffer, `lastClack` throttle (0.03s).
Exports: `unlockAudio()`, `clack(impact)` (0..1 input — caller normalizes), `thunk()`, `crackSnap()`,
`whoosh(power)`, `scratchSound()`, `shatterSound()`, `blip()`.

**Out of scope:** wiring any of it into main.ts (M1-C), enemy AI, waves, HUD.

**Acceptance:**
1. Floor (tsc + build).
2. Headless screenshot script (see `M0.md` recipe — vite preview + playwright + swiftshader): boot a
   scratch page or drive the existing app with `buildArena0` temporarily swapped via a `?level=arena`
   check ONLY inside your own test script (do NOT edit main.ts — build a tiny scratchpad harness page
   that imports your modules directly if easier). Assert visually (screenshot inspected): 4 pocket
   rims visible, rails with corner gaps, slab centered.
   Simpler accepted alternative: a node script asserting `sim.staticDefs`/`sensorDefs` counts and
   corner-gap geometry (rails' extents leave ≥ 1.2u openings at all 4 corners), plus one screenshot of
   whatever is bootable.
3. Audio: node-side smoke is impossible (WebAudio) — assert the module typechecks and that calling

## M1-C — Combat slice + HUD-min + boot rewire

**Files you own:** `v2/src/game.ts` (new), `v2/src/hud.ts` (new), `v2/src/main.ts` (rewire).

**Goal.** The rack loop: waves spawn, enemies think, physics events become cracks/chalk/stroke/SFX,
racks clear and escalate. This packet owns ALL game state; A/B built the parts, you compose them.

### hud.ts — class Hud, v1 method names, existing index.html IDs

`showHud(on)`, `setDepth(d, max)` (#depth), `setWave(w, total)` (#wave), `setChalk(c)` (#chalk,
`◆ ${c}`), `setStroke(v, max)` (#strokefill width %), `setCracks(taken, max)` (#cracks pip row,
class `lost` on spent pips), `banner(text, bad = false, ms = 1400)` (#banner, `.show`/`.bad`,
auto-clear timer). Port from v1 `/src/hud.ts` — combat methods only, no panels.

### game.ts — class Game (composition root for combat)

Constructor takes what main.ts already builds (sim, player factory hooks, hud, input, camera refs as
needed — your call on shape; main.ts becomes thin). State: `depth` (from `?depth=N`, default 1),
`stroke = CFG.combat.strokeStart`, `chalk = 0`, `cracks = 0`, `iFrames`, `banksThisShot`,
`sinksThisShot`, `grazedPockets: Set<number>`, `waveIdx = -1`, `waveCalm = CFG.combat.calmStart`,
`waveAge = 0`, `waves: Wave[]`, `boss: Entity | null`.

**Rack start** (`startRack(depth)`): reset per-rack state; seed = `dailySeed()`;
`rng = subRng(seed, RACK_TAG, depth)`; `seedAiRng((seed ^ AI_TAG) >>> 0)` once at boot.
`isBoss = depth === 8`, `isMiniBoss = depth === 4` (v2 depths — NOT v1's 12).
Boss rack: `waves = makeBossWaves(rng)`, spawn boss at normalized (0, −0.55) scaled by bounds, banner
`'THE 8-BALL'`. Mini-boss: `waves = makeMiniBossWaves(rng)`, spawn boss-kind at (0, −0.5), then
override `armor = 1`, collider radius ×0.82 & mass ×0.6 equivalent (spawn then adjust: simplest is
`spawnEnemy` then set `e.r *= 0.82` + rebuild collider radius via `e.collider.setRadius(e.r)` and
`e.collider.setMass(e.body.mass() * 0.6)`), banner `'THE RACK-MASTER'`. Normal: `waves =
makeRack(depth, rng, false)` (money racks are M2). `hud.setDepth(depth, 8)`.

**PhysicsEvents wiring** (replaces main.ts's M0 stubs):
- `onSink(e, pocketId)`: player → `scratch()` (below). boss kind → `audio.thunk()`; if `isBoss` →
  victory overlay (simple: banner `'RUN WON'` + reuse #overlay with a restart button — full flow M2);
  else banner `'RACK-MASTER DOWN'` + `rackCleared()`. Enemy → `despawn`, `audio.thunk()`,
  `sinksThisShot++`, `chalk += CFG.combat.sinkChalk + banksThisShot`,
  `gainStroke(CFG.combat.sinkStroke + banksThisShot * CFG.combat.bankStrokeBonus + (sinksThisShot > 1 ? CFG.combat.multiSinkBonus : 0))`,
  elite → banner `'ELITE DOWN'`.
- `onCollide(a, b, impact, tangent)`: `audio.clack(min(1, impact / CFG.combat.clackNorm))`. `pair` =
  the non-player of (a, b) (both non-player → return). Splitter check first: `pair.kind === 'splitter'
  && !pair.ai.didSplit && impact > splitImpact && (tangent > impact * splitTangentRatio || impact > splitHardImpact)`
  → `pair.ai.didSplit = true`, `despawn(sim, pair)`, `spawnSplitChildren(sim, pair)`. Crack check
  (only when the player IS one of the pair): `heavy = pair.kind === 'bumper' || pair.kind === 'boss'
  || (pair.kind === 'charger' && speedOf(pair) > chargerFastSpeed) || pair.elite`; if
  `(heavy && impact > crackHeavyImpact) || impact > crackLightImpact` → `takeCrack()`.
- `onSurface(e, impact, kind)`: `kind === 'wall' && e.kind === 'player' && speedOf(e) > bankMinSpeed`
  → `banksThisShot++`.
- `onGraze(e, pocketId)`: player only; once per pocket per shot (`grazedPockets`) →
  `gainStroke(CFG.combat.grazeStroke)`.
- `onFall(e)`: player → `scratch()`. Enemy → despawn + half sink reward: `chalk += 1`,
  `gainStroke(9)`, `audio.thunk()` (knocking enemies off the world is a legit v2 verb, masterplan).
  Boss-kind falling: treat as sink (same as pocket).
- `onPad`: no-op (M4).

**scratch()** — v2 semantics (LOCKED): `audio.scratchSound()`, respawn player at checkpoint (zero
velocities), `stroke = max(0, stroke - CFG.scratch.strokeCost)`, and `takeCrack(true)` ONLY if
`isBoss || isMiniBoss` (bypass i-frames gate for this forced crack but still play crackSnap). NOT a
run-ender. Banner `'SCRATCH'` (bad).

**takeCrack(force = false)**: gate `iFrames > 0 && !force` → return. `iFrames = CFG.combat.iFramesS`,
`cracks++`, `stroke = max(0, stroke - crackStrokeCost)`, `audio.crackSnap()`, `hud.setCracks`. At
`cracks >= maxCracks` → **shatter**: `audio.shatterSound()`, freeze play (phase 'dead'), #overlay
with `SHATTERED` + a 'RACK UP AGAIN' button that reloads (location.reload()) — full defeat flow M2.
Else banner `'CRACKED'` (bad).

**gainStroke(n)**: `stroke = min(CFG.combat.strokeMax, stroke + n)`; hud.

**fire wiring**: on `input.onFire` → reset `banksThisShot = 0`, `sinksThisShot = 0`,
`grazedPockets.clear()`, `audio.whoosh(power)`, then `control.fire(...)` as today.
First `pointerdown` anywhere → `audio.unlockAudio()` (once).

**Loop (own the timescale + wave logic in game.ts's update, called from main.ts's rAF):**
- Bullet time: while aiming (`aimShot != null`): `stroke > 1` → timescale `CFG.combat.aimTimescale`
  and `stroke -= CFG.combat.aimDrainPerSec * dt` (real dt); else timescale `aimTimescaleEmpty`.
  Not aiming → 1. (Replaces M0's flat 0.25.)
- Each frame: `iFrames -= sdt`; `updateEnemies(enemiesAlive, player, sdt, fx, bounds)` where sdt =
  scaled dt; `sim.checkGraze(player, pockets, events)`.
- Wave progression, v1 verbatim: `enemiesAlive()` = alive && kind !== 'player' && kind !== 'boss'.
  If none alive: boss-climax-open (boss alive && vulnerable) → nothing; else `waveCalm > 0` → count
  down by sdt, at ≤ 0 `spawnNextWave()`; else if `waveIdx >= 0` → `onWaveCleared()`; else
  `waveCalm = 0.5`. Anti-stall: NOT boss/miniBoss && enemies alive && `waveIdx ≤ waves.length - 2`
  → `waveAge += sdt`, at `≥ CFG.combat.waveStallS` force `spawnNextWave()`.
- `spawnNextWave()`: `waveIdx++`, `waveAge = 0`, `spawnWave(...)`, `hud.setWave(waveIdx + 1,
  waves.length)`, banner: any elite in wave → `'ELITE!'` (bad) else `'WAVE ${waveIdx + 1}'`.
- `onWaveCleared()`: boss/miniBoss → if `boss.armor > 0` `armor--`; if now 0 → `vulnerable = true`,
  banner `'ARMOR BROKEN — SINK IT'`, return; else banner `'ARMOR ${armor}'`,
  `waveCalm = calmBossBetween`. Normal → last wave? `rackCleared()` : (`waveCalm = calmBetween`,
  `gainStroke(CFG.combat.waveClearStroke)`).
- `rackCleared()`: `chalk += CFG.combat.rackClearChalk`, banner `'TABLE CLEAR'`, then after ~1.2s
  advance: `depth++` (cap 8 → after boss victory there's no advance), despawn all enemies, respawn
  player at spawn, `startRack(depth)` — the M1 escalation loop; the real route is M2.

### main.ts rewire

Keep: build stamp, boot, camera/render/input/preview wiring, grabRing, debug hook, stepSim shape.
Change: default level = `buildArena0` (M1-B); `?level=feel` keeps `buildLevel0` + M0's plain respawn
events so the M0 regression suite still runs against the old table (preview builder must match the
chosen level!). Game instance owns events/timescale/HUD. Extend the frozen debug hook additively:
`debug.spawn(kind, x, z, elite?)`, `debug.clearEnemies()`, `debug.combat() → { stroke, chalk,
cracks, wave, depth, enemiesAlive }`. `enterNode(kind)` stays a stub (M2).

**Out of scope:** money racks, drafts/shops/panels, route, powerups, markers, guardian-lip stat
(seam comment only), preview changes (M1-D — do not touch preview.ts/aim.ts).

**Acceptance:** floor + headless playwright drives (vite preview, `?debug`):
1. Boot arena: `debug.combat()` reports depth 1, wave 1 spawns after calmStart; wave counts match
   `makeRack` shapes.
2. `debug.clearEnemies()` twice (or thrice) walks waves → `'TABLE CLEAR'` → depth advances to 2.
3. `debug.spawn('skitter', 2, 2)` then scripted `debug.fire` toward it: sink registers (enemiesAlive
   drops, chalk +2, stroke +18 vs before).
4. `debug.spawn('bumper', ...)` in the player's path + hard fire into it → cracks becomes 1, stroke
   −15, second immediate collision does NOT double-crack (i-frames).
5. `debug.warp` off the edge → fall → respawn at checkpoint with stroke −30, cracks unchanged
   (depth 1 ≠ boss table).
6. `?depth=4`: mini-boss present with armor 1; `debug.clearEnemies()` → `'ARMOR BROKEN — SINK IT'`
   state (`vulnerable === true` via a debug read you add if needed); sinking mini-boss (fire or warp
   it over a pocket sensor via its body) does NOT produce the victory overlay.
7. `?level=feel` still boots the M0 table (M0 suite compatibility — orchestrator runs the old suite).

## M1-D — Preview honesty vs enemies

**Files you own:** `v2/src/preview.ts`, `v2/src/render/aim.ts`.

**Goal.** The aim ribbon must stop where the shot would first meet an enemy, and say so — v1 overlap-
tested its preview against live ball positions; same idea against Rapier-entity positions.

### preview.ts

- `PreviewResult` gains `hitEnemy: boolean` (default false each simulate).
- `simulate(...)` gains a 5th param: `obstacles: { x: number; z: number; r: number }[]` (live enemy
  positions+radii; caller passes them — preview stays registry-agnostic). After each ghost step,
  before recording: if `hypot(gx - ox, gz - oz) <= CFG.ballR + o.r` for any obstacle → record the
  point, set `hitEnemy = true`, break. (2D xz test — enemies and ghost share the floor; a ramp-borne
  ghost above an enemy is a miss, so also require `|gy - ballR-ish| < 0.5` … keep it simple:
  `abs(gy - (o.y ?? CFG.ballR)) < 0.6` if you take y in the obstacle struct — take y, it's free.)
- Do NOT add enemy bodies to the shadow world — the line stops at first contact by design (post-hit
  caroms would be a lie without simulating the enemy's response, dossier §preview-honesty).

### render/aim.ts

- `show(res, power)` — when `res.hitEnemy`: draw the existing dashes up to the endpoint, color the
  line the normal power color, and place a small impact marker at the endpoint: a ring sprite/mesh
  (RingGeometry ~0.12/0.18, color 0xffffff, opacity 0.9, facing up) that pulses subtly. Keep the
  constant-64-point buffer discipline (README/M0 lesson — never shrink a BufferGeometry attribute).
- `offWorld` (red) beats `hitEnemy` if both (a truncated-at-enemy line can't also be off-world —
  hitEnemy breaks first, so this is just: keep the existing offWorld branch first).
- Hide the marker in `hide()` and when `!hitEnemy`.

**Out of scope:** who computes the obstacles array (M1-C's main.ts passes
`all().filter(e => e.alive && e.kind !== 'player').map(e => ({ x, y, z, r }))` — coordinate via the
new signature only; if main.ts hasn't landed yet, update the CALL SITE in main.ts is M1-C's job —
your acceptance harness may drive preview directly).
